Bot commands

This page lists all of the commands this bot has to offer.

Public commands

The following commands are available for everyone to use. Keep in mind that, with the exception of !blacklisted and !commands, all of these can be disabled by typing !disablecommands in chat.

  • !blacklisted : Shows the current streamer's blacklisted maps (in case the streamer has any blacklisted beatmaps).
    Usage: !blacklisted
    Output: @<user> <blacklisted list link>
  • !commands : Shows the command list for this bot (this page).
    Usage: !commands
    Output: @<user> <command list link>
  • !roll : Rolls a random number between 0 and 100 (inclusive).
    Usage: !roll
    Output: <user>, you rolled <number> <emote>
  • !whatis : Searchs the specified term on Wikipedia.
    Usage: !whatis <term>
    Output: https://en.wikipedia.org/wiki/<term>
  • !whats : Searchs the specified term on Urban Dictionary.
    Usage: !whats <term>
    Output: https://www.urbandictionary.com/define.php?term=<term>
  • !nani : Searchs the specified term on Jisho (japanese dictionary). You can use non-unicode characters (e.g. Kanjis) to perform the search.
    Usage: !nani <term>
    Output: http://jisho.org/search/<term>
  • !fu : Flips the bird to someone/something. If no text is specified, you flip yourself.
    Usage: !fu <text>
    Output: Hey <text> azerFROST üñï
  • !np : Rick rolls people that want to know the currently playing map.
    Usage: !np
    Output: Current map: https://osu.ppy.sh/b/1033716
  • !maps : Baits people that expect the streamer's map collection.
    Usage: !maps
    Output: https://www.google.com/maps azerFROST
  • !mayday : Displays a message for streamers that are tired of getting the Mayday map requested.
    Usage: !mayday
    Output: Unfortunately, <streamer> has already played the Mayday map and won't play it right now. Thanks for requesting though! azerHappy
  • !maydont : Shows a clip of Azer talking about the Mayday map.
    Usage: !maydont
    Output: https://clips.twitch.tv/BreakableSecretiveJalapenoRitzMitz?tt_content=chat_card&tt_medium=twitch_chat
  • !bible : Displays a random bible verse out of a selected collection of verses. Source: http://www.sandersweb.net/bible/verse.php
    Usage: !bible
    Output: üôè " <verse contents> " - <verse name> üôè
  • !kms : Times you out for one second, only if the bot account is admin and your account is not admin or the broadcaster.
    Usage: !kms
    Output: @<user> 🔫 azerHappy
  • !yowzar : Displays the amount of times the user yowzar has talked in chats where the bot is active.
    Usage: !yowzar
    Output: yowzar has talked <amount> times in chat azerFROST

Streamer-only commands

The following commands are available only for the streamer to use.

  • !blacklist : Blacklists a map, which means users won't be able to request said map. You can use !bl as a shortcut. Keep in mind that the link has to be the one of a single difficulty. If you want to blacklist a set, you need to specify -set at the end.
    Usage for single difficulty: !blacklist <single difficulty beatmap link>
    Output for single difficulty: The beatmap <beatmap name> was blacklisted successfully.
    Usage for sets: !blacklist <single difficulty beatmap link> -set
    Output for sets: The beatmap set <beatmap name> was blacklisted successfully.
  • !whitelist : Whitelists a blacklisted map, which means users can request it again. You can use !wl as a shortcut. Keep in mind that the link has to be the one of a single difficulty. If you want to whitelist a set, you need to specify -set at the end. Of course, non-blacklisted maps can't be whitelisted.
    Usage for single difficulty: !whitelist <single difficulty beatmap link>
    Output for single difficulty: The beatmap <beatmap name> was whitelisted successfully.
    Usage for sets: !whitelist <single difficulty beatmap link> -set
    Output for sets: The beatmap set <beatmap name> was whitelisted successfully.
  • !changedisplay : Changes the way the requests are showed on osu chat. There's currently two modes:
    Requestor Last (default): The username of the requestor shows at the end of the request message.
    Requestor First: The username of the requestor shows at the beginning of the request message.
    By using this command, you toggle between the two of these.
    Usage: !changedisplay
    Output: Changed display type for requests to "<mode>".
  • !disablecommands : Disables all of the public commands with the exception of !blacklisted and !commands.
    Usage: !disablecommands
    Output: Command mode is now set to "No Additionals".
  • !enablecommands : Enables all of the public commands.
    Usage: !enablecommands
    Output: Command mode is now set to "All".

Administrator-only commands

The following commands are available only for the bot owner to use. These are useful if you are running the bot yourself by cloning the git repository.

  • !adduser : Adds a user to the users list. This means the bot will be available in the specified user's twitch chat. Keep in mind that the twitch username is NOT the same as the display name. Rather, it's the name that is present in the URL of the user's twitch channel. Also keep in mind that the osu! username is case-sensitive, and every whitespace (if any) has to be replaced with an underscore. For example, the osu! username Example Name would be written as Example_Name.
    Usage: !adduser <twitch username> <osu! username>
    Output: User <twitch username> has been successfully added.
  • !removeuser : Removes a user from the users list. This means the bot will no longer be available in the specified user's twitch chat. The same rules for !adduser apply here, except you don't have to type the osu! username.
    Usage: !removeuser <twitch username>
    Output: User <twitch username> has been successfully removed.
  • !checkstatus : Logs in the console the list of channels and blacklisted beatmaps for each channel in json format. This is useful if you have access to the console but not the filesystem.
    Usage: !checkstatus
    Output: <user list json array> <blacklisted list json object>
  • !transferbl : Transfers the blacklisted beatmaps from one user to another. This can be useful if a user has changed their account and they want to keep their blacklisted beatmaps on their new account, or if you just want an user to have the same blacklisted beatmaps as another user.
    Usage: !transferbl <twitch username from> <twitch username to>
    Output: The blacklisted beatmaps were transfered successfully.
  • !removebl : Removes all of the blacklisted beatmaps for the specified user. More specifically, it removes the user from the blacklisted list json object.
    Usage: !removebl <twitch username>
    Output: User <twitch username> has been removed from blacklists successfully.
